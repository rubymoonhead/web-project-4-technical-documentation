What is Hugo?

Hugo is a popular open-source static site generator written in the Go programming language.

What is a static site generator?

Per techradar, a static site generator like Hugo takes all the source files and creates a static website out of the source files.

A static site generator is a system that lets a website creator use certain code sections repeatedly and easily.

What is Go?

According to Golang.org, "Go is an open source programming language that makes it easy to build simple, reliable, and efficient software."

Wikipedia says that "Go is a statically typed, compiled programming language designed at Google in 2007 by Robert Griesemer, Rob Pike, and Ken Thompson.

Additionally, "Go is widely used in production at Google and in many other organizations and open-source projects."

Hugo's main selling point is that it's fast. It takes seconds to generate a site with lots of pages. Site generation speed prompted Smashing Magazine, a website that provides content and resources for web developers and web designers, to switch to Hugo from Wordpress.

Hugo has great Markdown support.

The Hugo site includes a table of content formats. In the table you'll notice that HTML and Markdown are supported.

What is Markdown?

Markdown is lightweight markup language. You can create formatted text using a plain-text editor.

"John Gruber and Aaron Swartz created Markdown in 2004 as a markup language that is appealing to human readers in its source code form.

Markdown is widely used in blogging, instant messaging, online forums, collaborative software, documentation pages, and readme files." Markdown is readable as-is without looking like Rich Text Format (RTF) or HTML.

We skimmed through the Markdown tutorials provided in the Hugo site. It appears it's easy to learn Markdown syntax in a single sitting. According to Markdown Guide "It doesn’t take long to learn the Markdown syntax, and once you know how to use it, you can write using Markdown just about everywhere."

Other Hugo features are that it can support dynamic API-driven content without plugins, has hundreds of pre-made templates, provides full support for multi-language sites, "allows you to output your content in multiple formats, including JSON or AMP, and makes it easy to create your own."

Skills for Implementing Hugo

Hugo might have a steep learning curve for people who have never coded and beginner web developers.

It would be best if you have basic knowledge of HTML, CSS, CLI (command line input) to install Hugo to your Windows, Linux, or Mac computer, know how and be willing to write your content with Markdown, and know how to use Git for content version control and to deploy your site.

How to Implement Hugo

If you plan to create your blog using Hugo, you must install Hugo on your computer. You can also install Hugo on Windows, macOS, and Linux.

We'll focus on installing Hugo on Windows 10 in this documentation.

After install, you must create a Hugo PATH, verify the executable, then create the Hugo site.

  1. Folder Setup

    The starting point for your new project will be C:\Hugo\Sites.

    C:\Hugo\bin will store executable files.

    • In Windows Explorer create a new folder: C:\Hugo. You can set up Hugo anywhere in your computer. For now we'll stay with C:\.
    • Create a bin subfolder in the Hugo folder: C:\Hugo\bin.
    • Create a Sites subfolder in Hugo: C:\Hugo\Sites.

  2. Download and Extract Hugo

    • Go to the Hugo Releases page.
    • Click on the latest release. The release has a "latest" button. Go down the list and choose the applicable Windows zip file to download.
    • We're downloading the hugo-0.89.4_Windows-64bit-zip.
    • Download the zip file into your C:\Hugo\bin folder.
    • Extract the zip file into the same C:\Hugo\bin folder. You should see three new files: The Hugo executable (hugo.exe), LICENSE, and README.md.

  3. Create a Hugo PATH

    You must add Hugo to your Windows PATH settings.

    • Right click on the Start button.
    • Click on System.
    • Click on Advanced System Settings on the right.
    • Click on the Environment Variables… button on the bottom.
    • In the User variables section, find the row that starts with PATH (all caps).
    • Double-click on PATH.
    • Click the New… button.
    • Type in the folder where hugo.exe was extracted, which is C:\Hugo\bin. The PATH entry should be the folder where Hugo lives and not the binary.
    • Press Enter when you’re done typing.
    • Click OK at every window to exit.

  4. Verify Hugo executable

    • Open a Command Prompt.
    • Type hugo help and press the Enter key.
    • You should see output that starts with:
      Hugo is the main command, used to build your Hugo site.
      Hugo is a Fast and Flexible Static Site Generator built with love by spf13 and friends in Go.
    • If you see something similar to the above output, then the installation is complete.
    • If you don’t, double-check the path that you placed the hugo.exe file in and that you typed that path correctly when you added it to your PATH variable.
      If you're still not seeing the output find solutions in the Hugo discussion forum or go to the support category of the discussion forum and request help.
    • At the prompt, change your directory to the Sites directory:
      C:\Program Files> cd C:\Hugo\Sites
      C:\Hugo\Sites>

  5. Create a Hugo site

    • Run the command to generate a new site. For example, using namedsite.com as the name of the site:
      C:\Hugo\Sites> hugo new site namedsite.com.
      You should now have a directory at C:\Hugo\Sites\namedsite.com. Change into that directory and list its contents.

Go to Install Hugo for the comprehensive guide.

What's Next?

You can choose from a variety of Hugo themes and apply the theme to your site.

Download the current version of the theme and unpack the zip file in your themes/chosentheme namedsite.com folder.

Find the config.toml file, a configuration file that lets you set different options for the site, in your namedsite.com folder. Open the file with Notepad. Add theme = "chosentheme" below your blog's title. You can further customize your config.toml file later.

From the command line run hugo serve. Open http://localhost:1313 in your browser, and you should see your live site.

You'll see your site's home page. There will be a list of posts. Choose “Creating a New Theme”. Open the file content/post/creating-a-new-theme.md to revise the post. Once you save said file, your website will automatically update with revisions you've made. Create a new post by creating a new Markdown(file extension .md) file with a file name you prefer. Now you can focus on creating your content. Start writing your content.

If you want, you can deploy your site for free using Netlify or Now or GitHub Pages. Be sure to get your own domain name though.

Pros and Cons

Pros

Hugo is easy to learn and set up, and has great documentation.

Hugo has a growing active open-source community.

Hugo is lighter than Wordpress and Hugo is great for content managers/web developers who prefer simplicity over a barrage of extensions and plugins they feel they don't need.

Since Hugo is a static site generator and the static site generator runs on your personal computer, your site is not accessible to anyone but you. Your website loads faster, and there is no database behind your site that might slow down load times when a user is on your site.

With Hugo, your HTML files are rendered on your computer. You get to review your files locally before they're hosted in the HTTP server. Your HTML files aren’t generated dynamically. Site performance is faster. HTTP servers excel at sending files and serve a number of pages with a fraction of memory and CPU resource required for dynamic sites.

Faster websites can rank better on search engines, so Hugo benefits.

With a static website, you don't need provider-hosted servers. You can use web developer platforms like Netlify, Now, or GitHub pages to host your Hugo site.

Cons

Hugo is not like a Wordpress site where you can edit content in a WYSIWYG admin window.

Hugo's learning curve might be steep for some since you must understand the basics of Markdown, some coding, and Git.
